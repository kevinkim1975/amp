# User Journey - AutoMed Proposal Generator

---

## Persona 1: 컨설턴트 (System User)
Goal: 30분 이내에 오류 없는 제안서 URL을 배포한다.

| 단계 | 행동 | 시스템 반응 | 감정/Thinking |
|------|------|-------------|---------------|
| 1. 준비 | 병원별 데이터를 수집해 hospital.json 작성 | - | "데이터 빠짐없이 입력했나?" |
| 2. 로컬 빌드 | npm run build 실행 | JSON 스키마 검증 -> 통과 시 페이지 HTML/JS 생성 | "오류 없네, 다행!" |
| 3. 오류 발생 (조건부) | 스키마 오류 발생 시 | 빌드 실패 + 구체적 오류 로그 | "필수 필드 누락? 바로 수정해야지." |
| 4. Git Push | git add . && git commit && git push | GitHub 업로드 완료 | "이제 자동으로 배포되겠지." |
| 5. 배포 모니터링 | Vercel Dashboard 확인 | 빌드 & 배포 진행, 완료 시 UUID URL 표시 | "끝났다! 링크 복사!" |
| 6. 전달 | Slack/이메일로 URL 전송 | - | "30분도 안 걸렸네." |

## Persona 2: 병원 의사결정권자 (End Viewer)
Goal: URL에 접속해 모든 페이지를 이탈 없이 열람한다.

UX 설계 수준: 초보자
- 대상: 디지털 경험이 제한적인 병원 경영진 (40~60대 다수)
- 원칙: 직관적 UI, 최소 클릭, 명확한 버튼 레이블, 별도 학습 불필요

| 단계 | 행동 | 시스템 반응 | 감정/Thinking |
|------|------|-------------|---------------|
| 1. 링크 수신 | 컨설턴트로부터 URL 수신 | - | "뭔가 새 제안서 링크구나." |
| 2. 접속 | URL 클릭 | 첫 페이지 로드 (<2s), 전문적 디자인 노출 | "깔끔하네, 믿음직하다." |
| 3. 탐색 | 네비게이션(다음/이전, 목차) 사용 | 즉시 해당 페이지로 스크롤 이동, 페이지 번호 업데이트 | "필요한 정보만 빠르게 볼 수 있군." |
| 4. 데이터 확인 | 차트/테이블 hover/클릭 | 인터랙티브 툴팁/하이라이트 | "숫자가 명확하네." |
| 5. 전 페이지 열람 | 마지막 페이지까지 확인 | 네비게이션이 끝 표시 | "전체 구조가 이해됐다." |
| 6. 의사결정 | 내부 회의/검토 | - | "이 제안 좋다, 진행해 보자." |

## Edge Journey: 모바일 열람
- 사용자는 스마트폰에서 URL 클릭 -> Tailwind 반응형 레이아웃으로 세로 스크롤 형태로 전환.
- 차트는 가로 스크롤 or 축소 레이아웃로 대응.
- 페이지 길이가 길어지는 대신 텍스트/그래프 가독성 유지.

---

## Moments of Truth
1. 빌드 시 JSON 오류 여부 - 컨설턴트의 신뢰 확보
2. 첫 페이지 로딩 속도 - 경영진의 첫인상 결정
3. 차트 인터랙션 부드러움 - 데이터 이해도 결정
